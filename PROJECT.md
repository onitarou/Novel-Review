# DESCRIPTION
このプロジェクトは、数十から百名程度の規模のサークル内という少数の限られた読者に対して、小説作品を公開・批評を受けるための個人向けwebサイトに関するものである。

# 利用者
投稿者と読者の二種類が存在する。
## 投稿者
現時点では管理者のみを想定している。
主に以下の操作が可能
- 作品の投稿
- 作品の編集
- 作品の削除
- 作品の共有URLの発行
- 作品の共有URLの停止・再発行
- 話ごとの公開状態の管理
- コメントの確認・状態管理
- 作品本文とコメントのバックアップエクスポート
- 作品の版管理・改稿履歴の確認

## 読者
管理者である投稿者から共有されたURLを使用して作品を閲覧できる。
主に以下の操作が可能
- 作品の閲覧
- 作品へのコメント
- 自分が投稿したコメントの編集・削除
- 本文表示に関する読書設定の保存

# 機能
## 作品管理
作品の管理は投稿者のみが行うことができる。
- **新しい作品の作成**：新しい作品のタイトルを決定し、本文の投稿先の箱として扱う
- **章の作成**：作品内部の話を章という単位で分類可能にすること。
- **話の投稿**：話ごとのタイトルを指定して、作品の編集領域から本文の執筆を行う。また`.txt`ファイルを読み取ることも可能とすること。
- **あらすじ**：作品には任意であらすじの文章を1000文字内で付与できること。
- **作者メモ**：作品には任意で作者メモを付与できること。作者メモには、批評してほしい観点、執筆上の補足、読者への注意事項などを記載できること。
- **話ごとの公開状態**：各話には公開状態を設定できること。少なくとも`下書き`、`限定公開`、`非公開`の状態を扱えるようにすること。
  - `下書き`：投稿者のみが閲覧できる編集途中の状態。
  - `限定公開`：作品の共有URLを持つ読者が閲覧できる状態。
  - `非公開`：投稿者のみが閲覧でき、完成済みであっても読者には表示されない状態。
- **本文記法**：本文内ではルビ・傍点などの小説向け簡易記法を扱えること。ルビは`{振り仮名}{ふりがな}`のように、対象文字列を前段、ふりがなを後段に置く形式を採用すること。
- **版管理・改稿履歴**：初回公開時の本文を`ver1`とし、以降は編集後に再公開操作を行うたびにバージョンをインクリメントすること。過去の版と改稿履歴を投稿者が確認できるようにし、コメントがどの版の本文に対するものか追跡できること。

## 表示
### 作品一覧ページ
作品一覧は投稿者のみが確認できる。全ての作品を羅列し、クリックすることによって各作品のページへジャンプすること。
### 作品ページ
投稿者と、作品固有のURLを共有された読者のみが閲覧できる。ただし、読者の視点では、公開状態が`限定公開`ではない話は表示されないようにすること。
作品ページにはその作品の章と話、あらすじ、作者メモを表示する。
- 左サイドバーを設け、ここに話を並べて表示すること。この表示は話ページでも維持される。
- 章が設定されている場合は左サイドバー上で章の区分を表示すること。
- 左サイドバー上の話がクリックされたとき、その話のページにジャンプすること。
- どの話も選択されていないとき、メインの領域には作品のあらすじを表示すること。
- 作者メモが設定されている場合は、あらすじとは区別して表示すること。
### 話ページ
本文を表示する。投稿者と、作品固有のURLを共有された読者のみが閲覧できる。読者には、話の公開状態が`限定公開`であるものだけを表示すること。
- 左サイドバーに話一覧が表示されたままにすること。
- メイン領域に話のプレビュー表示を行うこと。
- 縦書き表示と横書き表示の両方に対応すること。
- 縦書き表示・横書き表示、文字サイズ、行間などの読書設定を読者のブラウザ単位で保存できること。
- 横書き表示時、右サイドバーを設ける。この場所に読者がコメントを作成できるようにすること。
- 縦書き表示時、メイン領域の下に別の領域を設ける。この場所に読者がコメントを作成できるようにすること。
- コメント作成の際、本文の一部を選択した状態でコメントを作成すると、その文章を引用できるようにすること。
- コメントを選択したとき、そのコメントに対応する本文位置をハイライト表示できること。

### コメント一覧ページ
コメント一覧は投稿者のみが確認できる。作品横断または作品単位でコメントを一覧できること。
- コメントの未読・確認済み・対応済み・保留などの状態で絞り込めること。
- 作品、章、話、読者名、作成日時、コメント状態で絞り込み・並び替えができること。
- コメントから対象の話ページおよび引用位置へジャンプできること。

## コメント
読者は共有URLから閲覧できる話に対してコメントを作成できる。
- コメントには本文、対象作品、対象話、対象本文の引用、作成日時を記録すること。
- 本文の一部を選択してコメントした場合、その本文範囲を引用として保持すること。
- 引用コメントは本文上の対応位置と関連付け、コメント選択時に話ページ上でハイライト表示できること。
- 読者は自分が投稿したコメントを編集・削除できること。
- コメントには`未読`、`確認済み`、`対応済み`、`保留`などの状態を設定できること。
- コメントの状態変更は投稿者のみが行えること。
- コメントは本文の版と関連付け、改稿後もどの版に対するコメントか確認できること。

## 作品共有
作品単位で共有URLを作成できるようにすること。
- 共有URLは投稿者が任意のタイミングで手動発行することを前提とする。
- 共有URLは投稿者が任意のタイミングで手動停止できること。
- 共有URLは投稿者が任意のタイミングで手動再発行できること。
- 共有URLが停止または再発行された場合、停止済みのURLでは読者が作品を閲覧できないこと。
- 共有URLを持つ読者に表示される話は、公開状態が`限定公開`の話に限定すること。

## バックアップエクスポート
投稿者は作品本文とコメントをバックアップとしてエクスポートできること。
- 作品本文は作品、章、話、版の情報を含めてエクスポートできること。
- コメントは対象作品、対象話、対象本文の引用、コメント状態、作成日時、関連する本文の版を含めてエクスポートできること。
- エクスポート形式は、本文については`.txt`またはMarkdown、コメントについてはCSVまたはJSONを想定すること。

# DB
本番のデータベースには`Turso`を用いる。
アプリケーションからは`@libsql/client`で接続する。

本番環境では以下の環境変数を設定する。
- `TURSO_DATABASE_URL`：TursoのデータベースURL
- `TURSO_AUTH_TOKEN`：Tursoの認証トークン
- `ADMIN_PASSWORD`：管理画面ログイン用パスワード。本番環境では未設定や初期値を許可しない。
- `APP_SECRET`：Cookie署名やCSRF検証に使う秘密値。本番環境では32文字以上の十分にランダムな値を必須とする。
- `PUBLIC_ORIGIN`：共有URLを絶対URLで表示するための公開オリジン。例：`https://novel-review.onrender.com`。未設定時は安全のため相対URLで表示する。

これらが未設定の場合は、ローカル開発用として`file:data/app.sqlite`を使用する。
移行時に既存のローカルSQLiteデータをTursoへ引き継ぐ必要はない。
初回起動時に必要なテーブルを作成し、空のDBであればサンプルデータを投入する。
